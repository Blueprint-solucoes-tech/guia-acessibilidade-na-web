# Tasks

Esta pasta guarda specs pequenas de implementacao para o fluxo de Spec-Driven Development do projeto.

Use `tasks/template.md` como base para novas tarefas.

## Regras

- Uma task deve representar uma unidade pequena de trabalho.
- Toda task deve ter escopo e nao objetivos.
- Toda task com interface deve incluir checklist de acessibilidade.
- Toda task com logica deve incluir casos de teste.
- Tasks devem ser compreensiveis sem depender de conversas externas.

## Tasks iniciais

- `001-document-current-accessibility-tools.md`
- `002-document-component-reference-model.md`

## Fluxo recomendado

1. Planning Agent cria ou ajusta uma task.
2. Implementation Agent implementa apenas a task atual.
3. QA Agent revisa contra criterios de aceite.
4. Accessibility Review Agent revisa criterios de acessibilidade.
